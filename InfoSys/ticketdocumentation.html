Ticket Documentation Template

Make sure to note the exact error message e.g. User got an Error 401 when they tried to login, User got
a blank screen with no error, etc.
• Make sure to attach a screenshot if possible
• Make sure to note vital information e.g. User was just able to login yesterday, Verified that the
user is working from home, Verified that the status of their Pulse secure is “connected” etc.
• Make sure to enumerate all the troubleshooting performed
• Make sure to note all activities e.g. Checked AD, Connected to user’s PC via Bomgar, Consulted
this issue to Leads, User requested to leave the ticket opened, etc.
• In the even your call got disconnected, make sure to call the user back and document it e.g. Call
got disconnected, Called the user back at 123456789, no answer, etc.
• Make sure to note what resolved the issue and if the issue is resolved
• In the event that you have to close a ticket, please make sure to note as to why you are closing
the ticket e.g. Closing ticket as per 3 Strike Rule, Closing ticket as discussed with the user,
Verified that the issue is resolved, hence closing ticket, etc.
• If you have to put a ticket to “Pending Status” for reasons like awaiting consultation or awaiting
resolution from Resolver team, or the user said they will call back and requested the ticket to
leave their ticket opened, make sure to note the reason why e.g. Awaiting resolution from
squirrel, leaving ticket to pending, etc.
• If the ticket needs to be routed make sure to note the Queue name e.g. As per SD Guide, if all
fails, routing ticket to XYZ Queue.. etc.
