Application Issues Network / Shared Drive Consideration and Mapping
Universe Session Not Connecting – “Attempting to Connect to Host” at the bottom OR Documentum RDP Sessions fail.
Consideration / Probing / Resolution:

   When was the last time they were able to access the drive without an issue or error?
o  This way, you will be able to identify if they have an access to the drive.
o  If they are accessing the drive for the 1st time, ask for the request number and check the status
   of the ticket, if they don’t have one, provide the catalog title from our SD Guide and refer them to
   pITstop.
   Exact error message and full path of the drive
o  This will enable you to easily identify the issue and troubleshoot appropriately and effectively
o  If they do not have the path, you can ask the user to ask their colleague (who is currently
   mapped to the drive and can access it) for a copy of the path so you can easily map the drive for the
   user.

How to map a Network Drive - KB667:

 • Go to Computer
 • Click on Computer Option
 • Map Network Drive

choice 1: use ad.mvwcorp.com\mvcidfs\ to locate the site drive by MARSHA code
choice 2: Use the Site Support tab of the SD Guide to locate the direct mapping drive
choice 3: Ask the user for the mapping details
choice 4: Ask to view someone's computer who has the mapping setup already. Gather info that way.

Note

Access to the folder doesn’t guarantee access to the entire drive or access the contents of a
specific folder, it is important to educate the user about type of access in submitting Network Folder Access.
